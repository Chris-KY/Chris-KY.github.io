Coupe de France de Robotique

Bachelor Project · National Robotics Competition · STM32 · C

C STM32 Embedded Color Detection Motors Robotics K-Means

Overview

  • Team project for the Coupe de France de Robotique, a national student robotics competition. Ran over 2 years. The robot had to detect cups on a table, identify their color, pick them up, place each one in the correct destination, and avoid hitting the opponent robot during the match.
  • All code was written in C on an STM32 at the lowest level possible. No libraries, no AI, no high-level abstractions. We read the hardware documentation to know what signals to send to each component to make it work.

1) Color Detection — The Hard Part

  • The robot needed to tell Red cups from Green cups using a low-level color sensor, with no camera and no AI.
  • The core problem: the sensor output depends on both the cup color and the conditions (distance to the cup, ambient lighting). A fixed threshold for "red" or "green" does not work reliably across all situations.
  • Tested several approaches. The solution that worked: scan all cups visible from the current position, collect all the color readings together, then run a simple K-means clustering (k=2) to split the readings into two groups. One group is Red, the other is Green. No fixed reference value is needed for either color.
  • This approach is robust because it works by comparing the readings to each other, not to a hardcoded constant. As long as both colors are present, the algorithm separates them correctly regardless of distance or lighting.

2) Motors and Control System

  • Programmed the motor control directly on the STM32: PWM generation, speed control, direction, and stopping conditions.
  • Also handled the gripper mechanism for picking up and releasing the cups.

3) Robot Decision Logic

  • Wrote the state machine that governs what the robot does at each step: scan, identify color, move to the cup, grab it, navigate to the target zone, release.
  • The logic is entirely rule-based. Each state has a clear condition to enter and a clear action to execute, with no learning or probabilistic reasoning.

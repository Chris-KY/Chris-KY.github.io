Python Developer at Airbus

February 2024 to August 2024 · Contract · Toulouse, France

Python Design Patterns Refactoring Linux Git VS Code Aerospace

Overview

  • 6-month contract at Airbus to refactor a large legacy Python codebase used by mechanical engineers.
  • Rewrote the code using design patterns, added unit tests, packaged it as an internal library, and built an end-to-end pipeline for the engineering team.

1) The Legacy Codebase

  • Large collection of Python scripts with no structure, no unit tests, and duplicated logic throughout.
  • Hard to read, hard to modify, impossible to test reliably.
  • First step: read and understand all the existing code to know what it was doing before changing anything.

2) Refactoring with Design Patterns

  • Chose the right patterns for each problem. Two main ones used:
  • Builder: to construct complex objects step by step without messy constructors.
  • Strategy: to swap algorithms or behaviors without changing the surrounding code.
  • Wrote unit tests alongside the refactored code to make sure behavior was preserved and future changes safe.

3) Internal Library and Pipeline

  • Packaged the refactored code as an internal Python library used by multiple teams at Airbus.
  • Talked with the mechanical engineers to understand their workflow, then built a full end-to-end pipeline covering data ingestion, processing, and output.
  • Engineers could use clean, standard interfaces instead of copy-pasting old scripts.

What I Did

  • Read and understood the full legacy codebase.
  • Chose the design patterns that fit the problems, then rewrote the code.
  • Wrote unit tests for the new code.
  • Packaged everything into an internal library.
  • Talked with the engineers, understood their needs, and built the pipeline for them.

Low-Level Autonomous Car

Bachelor Project · STM32 · C

C STM32 Embedded Sensors Control Systems Autonomous

Overview

  • Coded autonomous behaviors for a car directly on an STM32 microcontroller in C, starting from the lowest level: no external libraries, no operating system, no AI.
  • Everything was written from scratch: reading sensor data, defining the logic, writing the control algorithms, and driving the actuators.
  • The goal was to understand and implement each layer of an autonomous system, from the hardware signal up to the decision.

1) Autonomous Behaviors

  • Line following: the car reads infrared sensors on the ground and stays on a marked path by continuously adjusting the wheel speeds.
  • Obstacle avoidance: using distance sensors, the car detects objects ahead and steers around them before making contact.
  • Hit detection: on-board sensors detect a collision and trigger a recovery sequence.
  • Rule-based decisions: the car evaluates its sensor state and selects an action based on explicit conditions written in code, with no machine learning involved.

2) The Low-Level Approach

  • All code was written in C directly on the STM32. No hardware abstraction layers beyond what the chip requires, no pre-built drivers.
  • Configured peripherals (GPIO, timers, ADC, PWM) manually through registers to control motors and read sensors.
  • Writing at this level forces a clear understanding of what is actually happening on the hardware, not just what a library abstracts away.

What I Did

  • Configured STM32 peripherals directly in C: GPIO, timers, ADC, PWM.
  • Wrote the sensor reading logic for infrared and distance sensors.
  • Implemented the control algorithms for motor speed and steering.
  • Designed and coded the decision logic: line following, obstacle avoidance, hit detection, and recovery sequences.

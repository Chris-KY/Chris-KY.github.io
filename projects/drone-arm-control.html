Drone Arm Control

Personal Project · Arduino Nano 33 BLE Sense + Raspberry Pi 5 + MAVLink

Python MAVLink ArduPilot Raspberry Pi 5 Arduino Nano 33 BLE Drones Embedded Bluetooth

Overview

  • Personal project: control a drone using arm gestures, with no joystick or traditional controller.
  • An Arduino Nano 33 BLE Sense strapped to the forearm reads arm orientation via its built-in IMU, sends commands via Bluetooth LE to a Raspberry Pi 5 mounted on the drone, which then controls the flight via MAVLink / ArduPilot.

1) Hardware Setup

  • Arduino Nano 33 BLE Sense: strapped to the forearm as a DIY wristband. Contains a built-in 9-axis IMU (LSM9DS1) for reading arm orientation. Connects to the Pi via Bluetooth LE.
  • Raspberry Pi 5 mounted on the drone: receives the Bluetooth data from the armband and runs the flight control script.
  • Drone running ArduPilot flight stack, connected to the Pi via MAVLink.

2) How It Works

  • The Arduino Nano 33 BLE Sense streams arm orientation data (tilt up / tilt down) over Bluetooth LE.
  • The Raspberry Pi 5 receives this data and maps it to flight commands: arm up makes the drone go up, arm down makes it go down, and so on.
  • When a valid arm orientation is detected (arm raised → go up, arm lowered → go down), the Pi runs a Python script that sends the corresponding MAVLink command to the ArduPilot flight controller.
  • Used the DroneKit Python library (built on MAVLink) to send commands cleanly without low-level MAVLink message construction.

3) Demo

Drone going up and down controlled by arm movement.

4) Future Work

  • Replace the IMU wristband with an EMG armband (e.g. OYMotion gForce Pro Band) that reads the electrical activity of wrist muscles to recognize distinct hand gestures.
  • This would allow mapping specific gestures — fist, open hand, wrist rotation — to different flight commands (throttle, yaw, pitch), enabling much finer and more intuitive control beyond simple up/down.
